Intro

This is a third publication in the seria of building forecast sales system for an airline. It covers organisation and management challenges have been appearing in this project and solutions to solve them.

Executive summary

Current team experience challenges during growth of new tasks which brings overhead on management and developers team. That's lead to set of unhealthy symptoms of development process which should be fixed to avoid future issues in the software delivery.

At first, the classic set of scrum methods was offered to fix silo of project knowledge:
- formalisation of requirements by tickets in CRM with user stories in the kanban style
- daily stand-ups in chat by sharing what each team manger did, planning to do and blockers on his or her way
- meeting minutes after calls to track agreements, action points and responsible persons

At second, the set of measures were introduced to stabilise codebase and make sure we always have a stable version of the project ready to deploy for the client:
- setup and keep up-to-date README file with all steps to deploy and run the project
- close the master branch for writing and push all work via Pull Requests after code review working by gitflow in separate branches
- set-up Continues Integration to compile project and run automate tests to confirm stability of the project in the automated way (still in backlog)

Organisational flaws were offered to solve by following set of measures:
- fix changes in requirements and scope of the project within a sprint to let the team focus on task and deliver it
- setting up relationships with the both two key stakeholders and gather feedback on a regular basis
- formalize all requests from the client in tickets to confirm we are all on the same page
- regular rotation of team members over projects tasks to mitigate silos of knowledge and cope with a bus factor

Similar to our bodies which are experienced constant exposure to viruses and others threats to our health, our projects are constantly faced with changes and challenges.
Similar to like healthy person could pass through this without much issues, but a person with broken or suppressed immune system get ill, our projects pass through all challenges when everything is organised right and get bad when something within is going wrong.
That's why it is important to setup the right set of measures in management and development work since the beginning and evolve it with time when a new challenges have come.
